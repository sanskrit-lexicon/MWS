## Reconciliation with the previously published MW counts

[`SYNTHESIS.md`](../SYNTHESIS.md) publishes `L.` = 40,212 and meta = 69,603, both from a
**literal** tag match. This census classes a citation by its **siglum**, so a meta siglum
that happens to carry a locator stays with its siglum. All of MW contains exactly
**2** such citations — `<ls>L. i</ls>` and `<ls>W. 1</ls>` — so the strata run +1 hedge / +1 authority against the
literal counts (hedge 40,213 = 40,212 + 1; authority 19,298 = 19,297 + 1). Immaterial to every
claim, recorded so a later session does not read it as drift.

One real drift **is** present: the record count is 286,525, against the 286,560 published in
`.ai_state.md` (2026-06-13, over `mw.txt` @ 2026-05-29). `mw.txt` has since lost 35 records to
upstream corrections. A08's artifact independently reports 286,525, so the current source,
not this parse, is what moved.

## Cross-check against A08's MW row (a measured divergence)

A08's committed artifact `data/obs/citation_registers.json` reports MW as
`entries` 286,525 · `ls` 312,160 · `lsWithLocator` 47,289 (15.15% locator-bearing).
The entry count reproduces exactly. The citation counts do not, for two reasons, both
verifiable against `mw.txt` and both making MW's apparatus look thinner than it is:

| | A08 | this census | why |
|---|--:|--:|---|
| Register-A citations | 312,160 | **320,828** | A08's extractor is literal — `_LS = re.compile(r"<ls>(.*?)</ls>", re.DOTALL)` ([`scripts/forensic/parse_cslorig.py:41`](https://github.com/sanskrit-lexicon/csl-atlas/blob/main/scripts/forensic/parse_cslorig.py#L41)) — so the **8,668** attributed citations never match |
| Locator-bearing | 47,289 | **60,820** | A08's locator rule is `DIGIT_RE = re.compile(r"\d")` ([`scripts/obs/citation_register_gaps.py:49`](https://github.com/sanskrit-lexicon/csl-atlas/blob/main/scripts/obs/citation_register_gaps.py#L49)): it cannot see the attributed form's locator (8,666 of the 8,668 bear one under this census's arabic-or-roman rule; 7,077 under a strict arabic-digit rule), and it scores the 4,866 plain-shape citations whose only locator is **roman** (`ŚBr. xiv`) as sourceless |
| Locator share | 15.15% | **18.96%** | |

**A definitional warning, so the two documents are not read as contradicting each other.**
A08's "bare abbreviation" means *no arabic digit* — one bucket holding everything that is
not a locator-bearing citation. On MW that bucket is 84.85% (264,871 of 312,160), and it
silently contains the `L.` hedge, `ib.`, `W.`, and every roman-locator citation. This
census's `bare` stratum (59.35%) means something narrower and load-bearing: *a named work,
no locator, and not a meta marker*. The two numbers answer different questions and neither
refutes the other; decomposing that one bucket is the point of this census. A08's **corpus-wide**
40.7% bare / 59.3% locator is a third thing again — a 44-dictionary aggregate dominated by PWG
(568,730 citations at 4.61/entry against MW's 1.09), and not a statement about MW at all.

So MW's linkable apparatus is **28.6% larger** than the published row records — a markup-shape artefact, not a scholarly disagreement. The direction of A08's finding is unaffected and its corpus-wide conclusion stands; the MW row wants regenerating. This is the same defect class as the `iti` rule A08 already corrected once (its space-or-quote rule undercounted KRM ~3× until a word-boundary rule replaced it).

MW's `iti` footprint here is **250** hits (0.0010/entry) against A08's 172 — both are noise-level, and both say the same thing: MW does not cite in the indigenous register. MW is a Register-A dictionary, exactly as A08 reports.

## Limitations

- The `iti` count is A08's **word-boundary proxy**, a register *indicator*: it fires on
  quoted running text and on grammatical/derivational `iti`, so it is an upper bound on MW's
  indigenous-register footprint, not a citation count.
- `attested` = **a locator is present**, an upper bound on linkability, not a verified link
  (A08 makes the same reservation for its 59.3% figure).
- Sigla here are locator-stripped surface forms, **not** normalised: `MBh.`/`MBH.` and
  `R.`/`Rām.` are not merged. Normalisation is csl-atlas' owned layer
  (`scripts/lib/source-siglum.mjs` + the curated alias table) and is deliberately not redone.
- Strata are per **citation occurrence**, not per sense: an entry mixing a hedged and an
  attested sense contributes to both.
